# 🎯 C++ 实习冲刺计划（8 周版）

## 你的真实起点

| ✅ 你有的 | ❌ 你缺的 |
|---|---|
| 真实的实习经历（中科创达） | 没有自己手写过完整 C++ 项目 |
| 3 篇论文 + 挑战杯奖（证明学习能力） | C++ 基础不扎实，理解浅 |
| 有 C++ 和 Linux 基础概念 | 简历写得太虚，面试一戳就破 |
| 会用 AI 写代码 → 说明代码阅读能力还行 | 没有**独立解决问题**的肌肉记忆 |

## 总体策略

> 面试要的不是天才，是要能干活、能沟通、肯学的人。

你需要做的就三件事：
1. **掌握 C++ 实习面试最高频的考点**（不用精通，够用就行）
2. **手写一个 300 行的小项目**（证明你有动手能力）
3. **学会怎么在面试里"诚实但自信"地展示自己**

## 详细计划

### 第 1 周：C++ 核心复习（每天 2-3 小时）

| 天数 | 内容 | 练习要求 |
|---|---|---|
| Day 1 | **智能指针**：`unique_ptr` / `shared_ptr` / `weak_ptr`、RAII | 手写一个 `MyVector` 简单版（用 RAII 管理内存） |
| Day 2 | **STL 容器**：`vector` / `map` / `unordered_map` / `string`，增删查改复杂度 | 用 `vector` 手写一个学生管理系统（增删改查） |
| Day 3 | **多线程**：`thread` / `mutex` / `lock_guard` / `condition_variable` | 手写生产者-消费者模型 |
| Day 4 | **C++11 核心**：`auto` / `nullptr` / 范围 for / Lambda | 每个各写 3 个 demo |
| Day 5 | **面向对象**：继承、多态、虚函数、`override` / `final` | 写一个 Shape 继承体系（形状求面积） |
| Day 6 | **文件操作**：`fstream` / `sstream` / `filesystem` | 写一个日志文件读取 + 解析程序 |
| Day 7 | **复习 + 刷题** | LeetCode 简单 3 道（用 C++，强制用智能指针） |

### 第 2 周：手写小项目（每天 3-4 小时）

**项目名称：简易 HTTP 服务器**

为什么要做这个？因为你的实习经历里写了 cpp-httplib，面试官很可能会问 HTTP 服务端原理。如果你说你写过一个小型的，面试官会非常认可。

**功能要求（约 250-350 行）：**

支持：
1. 监听端口，接受 TCP 连接（用 C++ socket API，别用库）
2. 解析 HTTP GET 请求（解析 method + path）
3. 返回静态文件（读取硬盘上的文件并返回）
4. 返回 404 处理
5. 多线程处理多个客户端（每来一个连接 new thread）

千万不能用任何第三方库，从 `socket()` `bind()` `listen()` `accept()` 自己写。

写完这个项目后，你在面试里的说辞就变成了：
> "我在实习中深入了解了 HTTP 服务端的工作原理，回来后自己用 C++ socket API 手写了一个简易 HTTP Server，进一步加深了对 TCP 网络编程的理解。"

这句话比你简历里吹的"高并发"强 100 倍。

### 第 3 周：面试高频知识点（每天 2 小时）

| 主题 | 具体内容 |
|---|---|
| **内存管理** | 栈 vs 堆、new/delete 原理、内存泄漏怎么排查（valgrind） |
| **编译链接** | 编译四步骤、静态库 vs 动态库、CMake 基本语法 |
| **Linux 基础** | 进程 vs 线程、gdb 调试、Makefile 读写 |
| **网络基础** | TCP 三次握手/四次挥手、阻塞 vs 非阻塞、select/epoll（了解概念） |
| **设计模式** | 单例模式（线程安全写法）、工厂模式、观察者模式（了解即可） |

### 第 4 周：刷题 + 复盘（每天 2-3 小时）

- LeetCode **简单** 20 道 + **中等** 10 道
- 重点题型：数组/字符串/链表/二叉树/哈希表
- 每道题用 C++ 自己写，不要 AI 帮助

### 第 5 周：准备面试话术 + 改简历

针对你的实习经历，准备好以下问题的回答：

**Q: 你在中科创达主要做了什么？**
> 我参与了一个智能座舱大模型评测平台的后端开发，主要负责 API 的调试验证、CMake 构建系统维护、以及前端联调。这个项目是用 C++17 写的，基于 cpp-httplib 实现 HTTP 服务。我在这段实习里第一次接触到完整的 C++ 服务端项目，也意识到自己基础还不够扎实，所以回来又自己写了一个简易 HTTP Server 来加深理解。

**Q: 这块的并发是怎么处理的？**
> 这个项目里我主要负责功能模块的部分，并发这块是资深同事负责的。基于我对 HTTP 协议的理解，我理解 cpp-httplib 底层应该是多线程处理请求的——每来一个请求创建一个线程或者用线程池。我自己写的小 Server 用的是每连接一个线程的模型。

**Q: ffmpeg 那块你怎么做的？**
> 我用的是 ffmpeg 的命令行工具，通过 `popen` 调用的。具体参数是同事配置好的，我负责调试验证不同场景下的抽帧效果。通过这个过程我理解了视频抽帧的基本原理。

### 第 6-8 周：投递 + 面试实战

- 每天投 5-10 家公司（BOSS直聘、实习僧、官网投递）
- 每次面试后复盘，记录不会的问题，查漏补缺
- 持续刷题保持手感
