使用auto声明的变量必须要进行初始化，以让编译器推导出它的实际类型，
在编译时将auto占位符替换为真正的类型。
auto z = 'a';       // z 是字符型 char
auto nb;            // error，变量必须要初始化
auto double nbl;    // 语法错误, 不能修改数据类型   

当变量不是指针或者引用类型时，推导的结果中不会保留const、volatile关键字
当变量是指针或者引用类型时，推导的结果中会保留const、volatile关键字

解释：
如果右边的原始变量带有 const（只读护盾），但是你用最朴素的 auto 去接收它（此时变量既没有配 * 也没有配 &），auto 就会直接把它的只读护盾（const）无情剥离掉！
一旦你显式地给 auto 焊上了引用（auto&）或者指针（auto*），这就意味着你不打算复制数据了，你是想直接去操控右边那个变量的物理真身（或者是抓取它的物理内存地址）！

❌ 限制一：绝对不能用于“非静态非 const 成员变量” （类大楼的图纸危机）
物理真相：C++ 在编译类定义的时候，必须明确地知道这个类实例化后，在物理内存里到底要占多少个字节（比如 int 占 4 字节，double 占 8 字节）。

❌ 限制二：绝对不能用于“函数形参” （C++20之前，你正在用的C++17铁律）

❌ 限制三：绝对不能用于“自动推导数组类型” （无法自动印刷数组）
物理真相：C++ 继承了 C 语言老祖宗的硬核规定，数组名在绝大多数右值表达式中都会自动退化（decay）为指针。auto 没能力违背这个底层物理规则，所以它无法帮你全自动印刷出一个全新的数组实体。

❌ 限制四：无法初始化没有明确初始值的变量 （打字机的断电惩罚）

decltype 是“declare type”的缩写，意思是“声明类型”。decltype的推导是在编译期完成的，它只是用于表达式类型的推导，并不会计算表达式的值。

decltype(表达式) 触发的是纯编译期行为。也就是说，编译器仅仅是去符号表里偷窥了一下括号里那个表达式的类型，压根就不会在运行时去真正执行括号里的代码！

基于范围的 for 循环是 C++ 告别传统越界 Bug 的安全宣誓。在面对大厂的高性能代码时，你的肌肉记忆应当是：零拷贝『遍历大对象用 const auto&，要改原值用 auto&；如果需要局部临时变量，直接在 C++17 的分号前面原地初始化它！


对应基于范围的for循环来说，冒号后边的表达式只会被执行一次。在得到遍历对象之后会先确定好迭代的范围，基于这个范围直接进行遍历。如果是普通的for循环，在每次迭代的时候都需要判断是否已经到了结束边界。

野指针（没有明确指向的指针，操作也这种指针极可能导致程序发生异常）

① 它能全自动变成任何类型的指针，但坚决不变成数字
nullptr 可以非常丝滑地赋值给任意类型的指针（不管是视觉图像指针、还是底层的 DDS 通信结构体指针），因为它在底层能隐式转换成任意指针类型

Lambda
[] - 不捕捉任何变量
[&] - 捕获外部作用域中所有变量, 并作为引用在函数体内使用 (按引用捕获)
[=] - 捕获外部作用域中所有变量, 并作为副本在函数体内使用 (按值捕获)
拷贝的副本在匿名函数体内部是只读的
[=, &foo] - 按值捕获外部作用域中所有变量, 并按照引用捕获外部变量 foo
